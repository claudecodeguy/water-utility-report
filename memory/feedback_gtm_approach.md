---
name: GTM-first decision making
description: User wants decisions optimized for fastest path to market, not over-engineering
type: feedback
---

Prioritize GTM speed when recommending next steps. Don't propose intermediate steps (like seeding mock data) that don't directly advance toward a shippable product. Always ask: does this step get us closer to real users faster?

**Why:** User is building toward launch and expects Claude to own the roadmap judgment, not just execute tasks.

**How to apply:** Before suggesting a step, ask "does this move the needle toward launch?" If it's scaffolding for its own sake, skip it. Flag tradeoffs when shortcuts have real costs, but default to the lean path.
